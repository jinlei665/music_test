# 歌手页面重构设计方案

> 日期: 2026-04-16
> 状态: 已批准

---

## 1. 概述

重构歌手列表页 (singerlis.html) 和歌手详情页 (singer_detail.html)，采用信息丰富型左右分栏布局，与现有 index.html 深色主题保持一致。

---

## 2. 歌手列表页 (singerlis.html)

### 2.1 页面结构

```
┌──────────────────────────────────────────────────────┐
│  导航栏 (top-nav + sub-nav)                          │
├──────────────────────────────────────────────────────┤
│  页面标题: 歌手列表                                   │
├──────────────────────────────────────────────────────┤
│  搜索栏 [🔍 输入歌手名搜索...]                        │
├──────────────────────────────────────────────────────┤
│  分类标签: [全部] [华语] [欧美] [日韩] [其他]         │
├──────────────────────────────────────────────────────┤
│  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐                 │
│  │头像 │  │头像 │  │头像 │  │头像 │                 │
│  │名字 │  │名字 │  │名字 │  │名字 │                 │
│  │歌曲数│  │歌曲数│  │歌曲数│  │歌曲数│                 │
│  └─────┘  └─────┘  └─────┘  └─────┘                 │
│  (3-4列网格卡片)                                     │
└──────────────────────────────────────────────────────┘
```

### 2.2 组件说明

| 组件 | 描述 |
|------|------|
| 导航栏 | 复用 index.html 的 top-nav + sub-nav 样式 |
| 搜索栏 | 实时过滤歌手列表，300ms 防抖 |
| 分类标签 | 按歌手地区/语言分类，点击筛选 |
| 歌手卡片 | 歌手头像(圆形) + 名称 + 歌曲数量 |

### 2.3 交互

- **点击卡片** → 跳转 `singer_detail.html?singer=xxx`
- **悬停卡片** → 显示半透明遮罩 + "进入主页"图标，轻微上浮

## 3. 歌手详情页 (singer_detail.html)

### 3.1 页面结构

```
┌─────────────────┬────────────────────────────────────┐
│                 │  [筛选排序工具栏]                   │
│   歌手封面      │  全部 | 专辑 | 单曲 | MV            │
│   (大图)        │  排序: 默认 | 最新 | 最热           │
│                 ├────────────────────────────────────┤
│   歌手名称      │  歌曲列表                           │
│                 │  01. 歌曲名 - 歌手  ▶ ⿶           │
│   收藏按钮      │  02. 歌曲名 - 歌手  ▶ ⿶           │
│                 │  ...                               │
│   歌手简介      │                                     │
│   (可折叠)      │  [加载更多]                         │
│                 │                                     │
│   相关推荐      │                                     │
│   (专辑卡片)    │                                     │
├─────────────────┴────────────────────────────────────┤
│              底部播放器 (rank.css)                    │
└──────────────────────────────────────────────────────┘
```

### 3.2 左侧固定区域 (约 30%)

| 组件 | 描述 |
|------|------|
| 歌手封面 | 200x200px 圆角图片 |
| 歌手名称 | 大字标题 |
| 收藏按钮 | 收藏该歌手所有歌曲，点击切换状态 |
| 歌手简介 | 可折叠，默认显示前3行 |
| 相关推荐 | 2-3 张专辑卡片，点击进入专辑详情 |

### 3.3 右侧滚动区域 (约 70%)

| 组件 | 描述 |
|------|------|
| 筛选标签 | 全部 / 专辑 / 单曲 / MV |
| 排序选项 | 默认排序 / 最新 / 最热 |
| 歌曲列表 | 可播放、可收藏、带序号 |
| 加载更多 | 分页或无限滚动 |

### 3.4 歌曲列表项

```
序号 | 封面 | 歌曲名 - 歌手名 | 专辑 | 时长 | 操作(播放/收藏)
```

### 3.5 交互

- **点击歌曲** → 调用 `load_music_by_id(songId)` 播放
- **点击收藏** → 调用收藏 API
- **点击专辑卡片** → 跳转专辑详情页

## 4. 设计规范

### 4.1 配色

| 元素 | 颜色 |
|------|------|
| 背景 | `#1a1a1a` 深色 |
| 卡片背景 | `#242424` |
| 悬停背景 | `#2a2a2a` |
| 主色调 | `#ec4141` (红色，与网易云一致) |
| 文字主色 | `#ffffff` |
| 文字次色 | `#999999` |
| 分割线 | `#333333` |

### 4.2 尺寸

| 元素 | 尺寸 |
|------|------|
| 卡片圆角 | 12px |
| 歌手头像 | 180x180px (列表页) / 200x200px (详情页) |
| 歌曲封面 | 50x50px (列表内) |
| 间距 | 20px (内边距) / 15px (卡片间距) |

### 4.3 字体

- 主字体: 系统默认字体
- 歌手名称: 18px, 粗体
- 歌曲名: 14px
- 歌手名(列表内): 12px, 灰色

### 4.4 图标

- 使用 Font Awesome 5
- 播放按钮: `fa-play`
- 收藏按钮: `fa-heart` / `fa-heartbeat`
- 更多操作: `fa-ellipsis`

## 6. API 接口

### 6.1 获取歌手列表

```
POST /content/find_singer
Response: { code: 200, data: [{ singer, singerimg, song_count }] }
```

### 6.2 获取歌手详情

```
POST /content/get_singer_songs
Request: { singer: "歌手名" }
Response: { code: 200, data: [{ id, song, singer, album, img, duration }] }
```

### 6.3 收藏歌曲

```
POST /users/add_favorite
Request: { songId, userId }
Response: { code: 200, action: "add"/"remove" }
```

## 7. 状态管理

- 登录状态: `sessionStorage.getItem('isLoggedIn')`
- 用户ID: `localStorage.getItem('userId')`
- 播放状态: `localStorage.setItem('playerState', JSON.stringify({...}))`
